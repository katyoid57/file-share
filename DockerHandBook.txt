## 導入時
1. smoで自分のアカウントを管理者権限にする

2. wsl をインストールして再起動する
wsl --install

3. Ubuntu をインストールする
// 利用可能なLinux配布を確認
wsl.exe --list --online
// インストール
wsl.exe --install Ubuntu-22.04
// ユーザ名とパスワードを設定 ※ユーザ名は英小文字、数字
Enter new UNIX username:
Enter new UNIX password:
// 起動確認
uname -a
// 最新化
sudo apt update
sudo apt upgrade -y

4. Docker Engine をインストールする
// 依存パッケージ
sudo apt install -y ca-certificates curl gnupg lsb-release
// Docker 公式 GPG キー追加
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
// リポジトリ追加
echo \
"deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
https://download.docker.com/linux/ubuntu \
$(lsb_release -cs) stable" | \
sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
// Docker Engine + Compose pluginをインストール
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
// Docker を sudo なしで使えるようにする
sudo usermod -aG docker $USER
// 設定反映
newgrp docker
// Docker デーモンを起動
sudo service docker start
// Docker デーモンの確認
docker --version
// docker compose の確認
docker compose version

5. Zscaler のルート証明書を WSL に追加する
① Windows で certmgr.msc を起動
② 信頼されたルート証明機関で"Zscaler Root CA" を探す
③ 右クリック → すべてのタスク → エクスポート
④ Base-64 encoded X.509 (.CER)を選択し、エクスポートファイルに名前を付ける　例：zscaler-root.cer（ダウンロードの中）
⑤ WSL に証明書をコピー
cp /mnt/c/Users/(ユーザ名)/Downloads/zscaler-root.cer ~/
// 確認
ls ~/zscaler-root.cer
⑥ 証明書を Ubuntu に登録
sudo cp ~/zscaler-root.cer /usr/local/share/ca-certificates/zscaler.crt
sudo update-ca-certificates
⑦ Docker を再起動してテスト
sudo service docker restart
docker run hello-world

7. GitHub CLIのインストール
// GitHub CLI 公式リポジトリを追加
type -p curl >/dev/null || sudo apt install curl -y
curl -fsSL https://cli.github.com/packages/githubcli-archive-keyring.gpg |
sudo dd of=/usr/share/keyrings/githubcli-archive-keyring.gpg
sudo chmod go+r /usr/share/keyrings/githubcli-archive-keyring.gpg
// apt リポジトリ登録
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/githubcli-archive-keyring.gpg] https://cli.github.com/packages stable main" |
sudo tee /etc/apt/sources.list.d/github-cli.list > /dev/null
// インストール
sudo apt update
sudo apt install gh -y
// 確認
gh version
// GitHub 認証
gh auth login
/* 質問の回答例

? How would you like to authenticate GitHub CLI? Login with a web browser
 -> その後、出てきたURLをブラウザにペーストし、ワンタイムコードを入力する。
*/
// 認証確認
gh auth status
/* 結果
✓ Authentication complete.
